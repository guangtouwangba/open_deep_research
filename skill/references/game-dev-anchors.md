# Game Development Domain — Authority Sources

## Game Design Theory
- GDC Vault (Game Developers Conference talks & postmortems)
- Jesse Schell《The Art of Game Design: A Book of Lenses》
- Raph Koster《A Theory of Fun for Game Design》
- MDA Framework (Mechanics, Dynamics, Aesthetics) paper
- Extra Credits (game design education channel)

## Game Engines
- Unity official documentation & best practices guides
- Unreal Engine official documentation & sample projects
- Godot official documentation & community tutorials
- Engine-specific GitHub issues and release notes

## Technical Art & Rendering
- Akenine-Möller《Real-Time Rendering》4th Edition
- GPU Gems series (NVIDIA)
- Shadertoy community examples
- GDC Technical Art track presentations

## Indie Game Development
- Steam Spy / SteamDB — sales data estimation
- Gamasutra/Game Developer magazine postmortems
- itch.io community data and trends
- Successful indie games' GDC postmortems

## Project Management
- Jason Schreier《Blood, Sweat, and Pixels》
- Agile/Scrum adaptations for game development
- Valve Handbook for New Employees

## Verification Rules
- Games must exist on Steam/App Store/itch.io
- Engine features must be in official documentation or changelog
- GDC talks must be findable in GDC Vault
- Sales data must cite SteamSpy/SteamDB or official announcements
